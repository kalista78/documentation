## Configuration Overview

Zyper uses environment variables and configuration files to manage settings across different environments.

<Info>
  Configuration priority: Environment variables > .env file > Default values
</Info>

## Essential Configuration

### Core Settings

```env
# Network Configuration
CHAIN_ID=999
CHAIN_NAME=HyperEVM
HYPEREVM_RPC_URL=https://lb.drpc.org/ogrpc?network=hyperliquid&dkey=YOUR_KEY

# Deployment Wallet
PRIVATE_KEY=0x... # Main deployer wallet private key
DEPLOYER_ADDRESS=0x... # Main deployer wallet address

# Multi-wallet System (Optional but recommended for production)
WALLET_1_PRIVATE_KEY=0x...
WALLET_2_PRIVATE_KEY=0x...
WALLET_3_PRIVATE_KEY=0x...
WALLET_4_PRIVATE_KEY=0x...
WALLET_5_PRIVATE_KEY=0x...
```

## Application Settings

### Server Configuration

```env
# API Server
NODE_ENV=production # development | production
PORT=3001
API_BASE_URL=https://api.zyper.xyz
FRONTEND_URL=https://zyper.fun

# CORS Settings
CORS_ORIGINS=https://zyper.fun,https://app.zyper.fun
CORS_CREDENTIALS=true

# Rate Limiting
RATE_LIMIT_WINDOW_MS=60000 # 1 minute
RATE_LIMIT_MAX_REQUESTS=100
```

## Configuration Files

### ecosystem.config.js (PM2)

```javascript
module.exports = {
  apps: [
    {
      name: 'zyper-api',
      script: './dist/services/ApiServer.js',
      instances: 'max',
      exec_mode: 'cluster',
      max_memory_restart: '1G',
      error_file: './logs/err.log',
      out_file: './logs/out.log',
      log_file: './logs/combined.log',
      time: true,
      env: {
        NODE_ENV: 'production',
        PORT: 3001
      }
    },
    {
      name: 'zyper-orchestrator',
      script: './dist/services/orchestrator/SimpleOrchestrator.js',
      instances: 1,
      max_memory_restart: '2G',
      cron_restart: '0 */6 * * *', // Restart every 6 hours
      env: {
        NODE_ENV: 'production'
      }
    }
  ]
};
```

## Security Best Practices

<Warning>
  Never commit sensitive configuration to version control!
</Warning>

<Steps>
  <Step title="Use Environment Variables">
    Store all sensitive data in environment variables, not in code
  </Step>
  
  <Step title="Rotate Keys Regularly">
    Implement key rotation for API keys and secrets
  </Step>
  
  <Step title="Encrypt Sensitive Data">
    Use encryption for stored credentials and wallet keys
  </Step>
  
  <Step title="Audit Access">
    Regularly audit who has access to production configuration
  </Step>
  
  <Step title="Use Secrets Management">
    Consider using AWS Secrets Manager, HashiCorp Vault, or similar
  </Step>
</Steps>
